# Workflows

Markdown SOPs that define objectives, required inputs, tools to call, expected outputs, and edge-case handling.

## Conventions
- One file per workflow, named descriptively (e.g., `scrape_website.md`).
- Each workflow should include:
  - **Objective** — what this workflow accomplishes
  - **Inputs** — what the agent needs before starting
  - **Steps** — ordered list of tool calls and decisions
  - **Outputs** — what success looks like
  - **Edge Cases** — known failure modes and how to handle them
- Keep workflows updated as you discover new constraints or better approaches.
